## 1. AI 요약 기능 (AI 플랫폼화 1번)
이미 쌓인 데이터(안축장·굴절·생활습관·케어 순응도)를 **부모용 자연어 요약**으로 풀어주는 기능. 분석 탭 최상단 카드.

### 설계 원칙 — "숫자는 앱이 계산, AI는 해설만"
- 백분위·증감·달성률은 클라이언트가 전부 계산해 **압축 컨텍스트**로 전달, AI는 그 숫자를 말로만 풀어줌
- 효과: 의료 도메인 안전(계산오류·환각 차단), 토큰 절감, 결정적 출력

### 구현
- `src/lib/aiReport.ts` — `buildReportContext()` 순수 계산 + 타입. `axialPercentile`·`compliance` 재사용
- `src/app/api/myopia-report/route.ts` — Solar 호출 라우트 핸들러(서버측 키 보관, `json_object` 모드)
- `src/components/analytics/AiReportCard.tsx` — 카드 UI: 생성 / 다시 생성 / **접기·펼치기**, 면책 문구 UI 고정
- `src/lib/supabase/queries.ts` — `fetchLatestReport` / `saveReport`
- `docs/sql/2026-06-23-ai-reports.sql` — `eyebody_ai_reports` 테이블 + RLS(`is_guardian`)

### 모델: Claude → Solar
- 초기엔 Claude Haiku 4.5로 설계했으나, 비용(최소충전·과금) 이슈로 **이미 호출 가능한 Solar로 전환**. OpenAI 호환이라 `fetch`로 연동(SDK 추가 없음), 모델·엔드포인트는 env로 오버라이드 가능

### 안전(아동 건강 도메인)
- 진단·처방·계산 금지, 또래 백분위는 쉽게, "안축장은 나이 들며 자연 증가" 균형 표현, 단정 회피, 필요 시 "전문의 상담" 연결
- 면책 문구는 AI가 아니라 UI에 **고정**(항상 정확하도록)

### 피드백 반영
- **"월간" 표현 제거** — 검사 주기가 자녀마다 달라 월 단위 cadence를 암시하지 않도록 제목 `AI 요약` + 프롬프트에서 "이번 달" 금지("최근 검사 기준")
- **카드 접기/펼치기** — 저장된 리포트는 기본 접힘(헤드라인만), 방금 생성한 건 펼침

## 2. Vercel 이전 (정적 → 서버리스)
AI 라우트 핸들러는 `output:'export'`(정적) 빌드에서 동작 불가 → Vercel 서버리스로 이전.

### 코드 변경
- `next.config.ts` — `output:'export'`·`basePath:'/MyoNote'` 제거(루트 도메인)
- `public/manifest.json`·`sw.js`·`SwRegister.tsx` — PWA 경로 `/MyoNote` → 루트 `/`
- `layout.tsx`·`settings` — 하드코딩 gh-pages URL → `NEXT_PUBLIC_SITE_URL`
- `.github/workflows/deploy.yml` 삭제(gh-pages 자동배포 중단, Vercel이 담당)
- `src/middleware.ts` 삭제(no-op, deprecated 경고 제거)
- 검증: `next build`에서 `/api/myopia-report`가 서버리스 함수로 정상 빌드

### 배포 (myonote.vercel.app)
- `main` 머지 후 Vercel 프로젝트 연결 → 자동 배포
- 환경변수: `NEXT_PUBLIC_SUPABASE_URL/ANON_KEY`, `NEXT_PUBLIC_SITE_URL=https://myonote.vercel.app`, `UPSTAGE_API_KEY`(서버 전용), `SOLAR_MODEL`
- 카카오 OAuth: Supabase Auth → URL Configuration에 새 도메인(Site URL + Redirect URLs) 등록

### 배포 중 만난 이슈 (해결)
- **루트 404** — 좋은 빌드가 Preview로 떠 있고 프로덕션은 옛 빌드를 가리킴 → 프로젝트 재생성/프로덕션 연결로 해결
- **카카오 로그인 NXDOMAIN** — Vercel env에서 `NEXT_PUBLIC_SUPABASE_URL` 값에 anon key가 들러붙어 Supabase 호스트명이 깨짐 → 변수 분리 + Redeploy(`NEXT_PUBLIC_*`는 빌드 시 bake-in)로 해결
- 최종: 로그인·자녀 프로필·기록·AI 요약 생성까지 프로덕션 정상 확인 ✅
