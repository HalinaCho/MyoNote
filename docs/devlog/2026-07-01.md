## 0. (오전) OCR 검사일 자동인식 안정화 — `f0b4614`
검사일 추출은 원래 파이프라인에 있었으나 `<input type="date">`가 `YYYY-MM-DD`만 받아, 다른 표기(`2024.03.05`·`2024년 3월 5일`·`24-3-5`·시각 포함)를 조용히 무시하던 문제.
- `examExtract.ts`에 **`normalizeExamDate()`** 신설: 정규식으로 연-월-일 3숫자 추출 → `YYYY-MM-DD`. 존재하지 않는 날짜(2/30)·미래·2000년 이전은 `null`로 폐기(깨진 값 저장 방지). node 15케이스 검증 통과.
- `route.ts` `examDate` description 강화: 생년월일·출력일·예약일이 아닌 **실제 측정일** 선택 + 형식 명시.

## 1. OCR 속도 3배 단축 (60초 → 20초)
검사지 1장 흐름: `사진 → downscaleImage(클라) → /api/exam-extract → Upstage IE → 필드`. 지연 지점을 진단해 두 축에서 개선.

### ① Vercel 함수 리전 = Seoul(icn1) — 최대 효과, 코드 무변
- 리전 설정이 아예 없어 기본값 **미국(iad1)** 이었음. 사용자도 Upstage API도 모두 한국인데 검사지 이미지(큰 페이로드)가 **태평양을 두 번 왕복**(사용자→미국 함수→한국 Upstage→미국→한국).
- Vercel 대시보드 → Settings → Functions → **Function Region을 Seoul(icn1)** 로 변경 → 전 구간이 한국 내 처리. **OCR·챗봇·요약·해설 4개 Solar 라우트 전부 수혜.**
- ※ `preferredRegion` 코드 export는 **Vercel에선 edge 런타임 전용**이고 구체 리전 코드도 안 받음. 큰 이미지 본문 때문에 Node 런타임이 필요하므로 **대시보드 설정이 유일 경로**(코드에 남지 않는 인프라 설정 — 재설정 시 유의).

### ② 클라이언트 이미지 디코드 최적화 — `cd9c2e6`
- `downscaleImage`가 `FileReader → base64 data URL → Image.src(거대 base64 재디코드) → canvas` 경로였음. 폰 사진 5~10MB에선 base64 왕복이 모바일에서 느리고 메모리도 큼.
- **`createImageBitmap(file)`** 빠른 경로로 전환 — blob을 직접 디코드해 base64 중간단계 제거. EXIF 회전(`imageOrientation: 'from-image'`) 반영으로 세로/가로 사진 안전. 구형 브라우저는 기존 `FileReader→Image` 폴백으로 자동 강등. `ImageBitmap.close()`로 메모리 즉시 해제.
- 출력 픽셀(2000px · q0.85) 동일 → **OCR 정확도 무손실.**

> **결과: 60초 → 20초.** ①이 지배적 요인, ②는 클라 처리·메모리 부담 경감.
> 남긴 손잡이: `downscaleImage`의 `maxDim`(2000). 더 줄이려면 1600으로 픽셀 ~36% 감소 가능하나 작은 글자 정확도 영향 → 미적용(실측하며 조정).

## 2. 20초 대기 UX ① — 단계별 안내 문구
정적 "인식 중…" 하나만 뜨던 대기 화면에, **가짜 퍼센트 없이** 진행 상태를 알리는 라이브 문구 추가(`records/page.tsx` `handleExtract` + 업로드 버튼 아래 안내줄, `aria-live`).
- **문구 3단계** — `51940be` 도입 후 노출 균형을 위해 시간 기반으로 재배분(`677ffec`):
  - 0~4초 `사진을 최적화하고 있어요…`
  - 4~13초 `AI가 검사값을 읽는 중이에요…`
  - 13초~완료 `거의 다 됐어요, 잠시만요…`
- 실제 단계에 묶지 않고 **시간 배분**한 이유: `createImageBitmap` 최적화로 downscale이 <1초라 첫 문구가 깜빡하고 사라졌음. 문구는 원래 안심용이라 시간 배분이 자연스러움.
- **스피너 중복 제거**(`4871d41`): 업로드 버튼이 이미 회전 스피너를 담당하는데 안내줄에도 회전 아이콘이 있어 스피너 2개가 동시에 돌아 산만 → 안내줄 아이콘 제거(모션 소스 = 버튼 1개).

## 3. 20초 대기 UX ② — 진행바 (시간 기반 트리클)
Upstage는 단일 응답이라 **실측 진행률이 없음** → 가짜 100%를 주장하지 않는 정직한 트리클 바로 구현(`6e571a2`). 문구 아래 teal 진행바.
- **경과시간 기반 로지스틱(S자)** 곡선. 반복 튜닝:
  - 최초: 지수 ease-out `92·(1−e^(−t/7))` → 초반이 가장 빨라 부자연스러움(`6e571a2`)
  - 수정1: 로지스틱 중점 10·기울기 0.4·캡 92 → 초반 너무 느리고 20초에도 90%에 못 미쳐 종료(`156a435`)
  - **최종**(`4d15a99`): **중점 8초·기울기 0.5·캡 96** → `96 / (1 + e^(−0.5·(t−8)))`. 4초 ~11%, 14초 90%+ 돌파, 완료 전 100% 미도달(정직). 응답이 20초보다 빨리 끝나도 높은 지점에서 마무리.
- 바가 모션을 담당하므로 문구의 `animate-pulse`는 제거(산만함 방지). 완료 시 바가 사라져 끝을 알림.
- 미세조정 손잡이: 초반 속도=중점(8), 끝 도달 시점=기울기(0.5), 끝 높이=캡(96).

## 4. 검사 탭 — 연도 네비게이션 → 연도별 접기 세로 스크롤
검사값이 연 2~4건뿐이라 연도 `<select>`로 전환하는 게 번거로웠음.
- 전체 기록을 **한 페이지에 세로 나열** + **연도별 그룹 접기/펼치기 토글 헤더**(연도+건수). 헤더는 **sticky**(bg `#edf7f6` 매칭)로 스크롤 중 현재 연도 유지. `selectedYear`/`filtered` 제거 → `collapsed` Set로 교체(`20dfc29`).
- **기본 펼침 = 최근 연도만**: 연도 증가 시 스크롤 부담↓. 최초 로드 1회 `useRef` 가드로 최근 연도만 펼치고 나머지 접음. 이후 사용자 토글·새 연도 추가는 존중(`f9b5016`).
