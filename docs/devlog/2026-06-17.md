### 1. 안축장 백분위 기능 추가 (`분석 > 안축장`)

#### 참조 데이터 및 계산 로직 (`src/lib/axialPercentile.ts` 신규)
- 동아시아 소아 코호트 기반 안축장 참조 데이터 (6~13세, P10/P25/P50/P75/P90)
  - 기존 MVP(Eyebody 프로젝트)와 동일한 수치 적용
- `calcAgeYears(birth, examDate)` — 검사 시점 나이(소수) 계산
- `calcPercentile(al, age)` — 밴드 보간법으로 백분위 산출 (P3~P97)
- `pctLabel(pct)` — 상위/하위/정상 범위 레이블 반환
- `normCurve(key)` — 차트용 스무스 참조선 포인트 배열 (나이 0.25 단위)

#### 또래 비교 차트
- X축: 나이(세), Y축: 안축장(mm) — 날짜 기반 → 나이 기반으로 전환
- P10 / P25 / P50 / P75 / P90 참조선 표시 (점선)
- P25–P75 구간 teal 채움 밴드
- 아이 실측 포인트(우안·좌안) 겹쳐 표시
- X축 범위: 오늘 기준 현재 나이 ±3세, 정수 눈금

#### 또래 비교 요약 카드
- 최근 검사 기준 OD/OS 백분위 표시
- 브랜드 teal(`#10bcad`) 컬러로 통일 — 상위/하위/정상 구분 없이 단일 색상
- PctBar: 0~100 트랙 위에 닷 마커로 백분위 위치 시각화

#### 뷰 구조 변경
- 기존: 변화 추이 / 또래 비교 토글
- 변경: 두 뷰를 한 페이지에 순서대로 통합 (토글 제거)

### 9. 카카오 OAuth 로그인 전체 수정

#### 문제 1 — KakaoLoginButton JSX 문법 오류 (빌드 실패 원인)
- `return` 내에 `<button>`, `<p>`, `<p>` 3개 요소를 fragment 없이 나열 → 빌드 자체가 실패
- `<>...</>` fragment로 감싸서 해결, 디버그 텍스트 제거

#### 문제 2 — Supabase 클라이언트 교체
- `@supabase/ssr` `createBrowserClient` (쿠키 기반) → `@supabase/supabase-js` `createClient` (localStorage 기반)
- 정적 사이트에서 쿠키 스토리지가 페이지 간 PKCE code verifier를 유실하는 문제 근본 해결

#### 문제 3 — auth/callback 이중 실행
- `useSearchParams` 의존성으로 effect가 2회 실행 → 첫 번째 `exchangeCodeForSession` 성공 후 URL이 바뀌면서 두 번째 실행이 auth_failed로 리다이렉트
- `useRef(false)` 가드로 1회 실행 보장
- `window.location.search`에서 직접 code 파라미터 추출 (useSearchParams 우회)

#### 문제 4 — Implicit flow 대응
- Supabase가 `#access_token=` 해시 방식(implicit)으로 응답하는 케이스 추가 처리
- `?code=` 있으면 PKCE → `exchangeCodeForSession`, 없으면 implicit → `onAuthStateChange` + `getSession` 대기

#### 문제 5 — 로그아웃 후 자동 로그인
- `prompt: 'login'` queryParam 추가 → 카카오 세션 살아있어도 항상 계정 입력 화면 표시

#### 수정된 리다이렉트 (basePath 누락 해결)
- `router.replace('/login')` → `window.location.replace(NEXT_PUBLIC_SITE_URL + '/login')`
- 적용 위치: `settings/page.tsx`, `DashboardShell.tsx`, `page.tsx`, `auth/callback/page.tsx`

- `SerTab`: Y축 tick `${v}D` → `${v.toFixed(1)}D` 부동소수점 오류 수정
- `AxialTab`: Y축 tick `${v}mm` → `${v.toFixed(1)}mm` 소수점 1자리 통일

---

### 19. 안축장/굴절도수 차트 가로 스크롤 + Y축 고정

- 8개 이하: 기존 동일 (차트가 컨테이너에 맞춤)
- 9개 이상: 포인트당 52px 확장, 가로 스크롤 가능, 기본 위치 = 최신 데이터(오른쪽 끝)
- **Y축 항상 표시**: Y축 전용 차트(52px 고정) + 데이터 스크롤 차트 분리
  - 왼쪽 캔버스: Y축만, `afterFit`으로 너비 52px 고정, 이벤트·애니메이션 없음
  - 오른쪽 캔버스: Y축 `display:false`, X축+데이터만, 스크롤
  - 두 차트 동일 yMin/yMax 공유 → 격자선 높이 일치
