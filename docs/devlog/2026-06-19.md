## 1. 분석 차트 Y축 정밀 개선

### 안축장 / 굴절도수 변화추이 (TrendView / SerTab)
- Y축 단위를 눈금에서 분리해 별도 라벨로: 안축장 `(mm)`, 굴절도수 `D`(눈금 인라인)
- min/max를 **0.5 단위로 스냅**(floor/ceil), `stepSize: 0.5`로 모든 눈금 정렬
- Y축 폭 축소(52→34/38px), 차트 높이 비율 `2:1 → 3:2`
- 굴절도수 눈금에 음수 부호(`-1.5D`) 복원

### 또래 안축장 백분위 비교 (PctView)
- Y축 눈금 `stepSize 0.5`(12개) → `1`(7개, 21~27 정수)로 밀도 완화
- `mm` → `(mm)` 괄호, 숫자 컬럼 위 정렬
- Chart.js v4 축 경계선 제거(`border: { display: false }`)
- 데이터 점 크기 축소(3→2), 연결선 얇게(1px)

## 3. 변화추이 차트 인터랙티브 수직선 + 값 박스 (안축장/굴절도수 공통)

### 기능
- 그래프에 **선택 지점 수직선** 표시(기본 최신 데이터)
- 다른 지점 터치 시 수직선과 값 박스가 해당 데이터로 **이동**(`interaction: { mode: 'index' }`)
- 값 박스: 선택 지점 x를 따라 이동, 수직선 끝과 연결, 날짜(1줄) + OD/OS 수치(2줄)

### 핵심 구현 / 트러블슈팅
- **수직선 이동 버그**: react-chartjs-2가 플러그인을 생성 시점 idx로 고정 → `idxRef`로 항상 최신값 읽도록 수정
- 수직선: `beforeDatasetsDraw`로 데이터 선 **뒤**에 렌더, 점선 + 스크롤바 그레이(`#9CA3AF`)
- 값 박스: **소프트 플로팅 섀도우**(뉴모피즘 시도 후 다듬음), 차트 상단 40px 여백으로 그래프와 겹침 방지
- 박스 `offsetWidth` 측정 후 절반값으로 **클램프** → 최신/최오래 날짜에서도 안 잘림

## 5. 캘린더 UI 정리

### 날짜 칸
- 정사각형(`aspect-square`) 유지, 날짜 숫자 확대 + 반응형(`text-sm sm:text-base`)
- 상태 아이콘(✓/−/✕) **제거** — 배경색(teal/amber/rose)으로 케어 상태 구분
- 생활습관 표시: 키·폰 아이콘 2개 → **기록된 날만 작은 회색 점 1개**(하단 행 높이 고정으로 숫자 위치 유지)

- 2단 분리 시도 후 → **한 줄 인라인 + 그룹 세로 구분선**으로 정리
- 좌: 케어완료 / 부분(케어 2개 이상일 때만) / 미완료, 우: 생활습관 점

# 추가 세션 (Opus) — 난시케어 커스텀 + 케어별 활성기간

## 7. 설정 페이지 정리
- 자녀 프로필 바텀시트 라벨 "진행 중인 케어" → **"진행 중인 난시케어"**
- 최상단 "현재 선택된 자녀" 카드 **제거**(자녀관리 카드와 정보 중복) → 자녀관리 행에 생년월일 추가(`만 N세 · YYYY-MM-DD`)

## 8. 난시케어 커스텀(하이브리드) + 케어별 활성기간 ★ 핵심

### 배경 / 문제
- 기존: 케어가 아트로핀·드림렌즈 **고정 2종**(불리언 컬럼)
- 케어 종류를 바꾸면(1↔2개, 휴약 후 재개) 캘린더 색상·홈 달성률이 **현재 설정 기준으로 과거를 재계산** → 과거 완료/부분/미완료가 **왜곡**되던 버그

### 설계 (JSONB, 정규화 테이블 대신)
- `eyebody_children.treatments jsonb`: 케어 정의 배열 `{key, name, preset, schedule, periods:[{s,e}]}`
  - 그 날짜 활성 = 기간 중 `s ≤ 날짜 ≤ (e ?? 오늘)`
- `eyebody_treatment_logs.done jsonb`: 그날 완료 케어 맵 `{key:true}`(임의 케어 지원)
- **JSONB 선택 이유**: 새 테이블마다 Supabase RLS 재작성 필요 → JSONB는 기존 RLS 상속. 데이터 전량 클라이언트 로드 후 JS 계산이라 정규화 불필요
- 구 컬럼(`treat_*`, `atropine/dreamlens`) **보존** + 읽기 폴백·쓰기 병행(점진 전환)
- Supabase 마이그레이션 SQL 직접 실행(컬럼 추가 + done/treatments 백필)

### 기능
- **하이브리드**: 프리셋(아트로핀 점안 / 드림렌즈) + **직접입력**(이름 + 스케줄)
- 케어별 **활성 기간** 자동 관리: 켜짐=기간 시작, 꺼짐=어제까지 마감(과거 보존), 재개=새 기간
- `getDayStatus/calcMonthCompliance/calcStreak`가 고정 배열 → **날짜별 활성 케어 함수**(`treatmentsForDate`)를 받도록 변경 → 캘린더·홈 7일·월간·연속 모두 날짜별 분모로 정확
- **캘린더 바텀시트**가 "그 날짜에 활성이던 케어"만 표시(5월=둘 다, 6월=현재만)
- 월간 달성률: 케어 없던 날은 분모에서 제외

### 트러블슈팅 / 후속 수정
- **제거 즉시 미반영(off-by-one)**: 제거 시 기간 종료를 "오늘"로 잡아 당일 여전히 활성 → **어제까지 마감**으로 수정(오늘부터 비활성). 같은 날 추가+제거 시 기간 삭제
- **+추가 버튼 레이아웃 넘침**: 스케줄 입력칸 `min-w-0` + 버튼 `flex-shrink-0`
- **케어 편집 UI 통일**: 프리셋(체크박스)/커스텀(휴지통) 불일치 → 진행 중인 케어를 **단일 목록 + 휴지통 버튼**으로 통일, 미추가 프리셋은 `+` 칩으로 분리
