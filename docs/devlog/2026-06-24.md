## 1. 검사 결과 즉시 해설 (AI 기능 2번)
검사기록 카드별 **"AI 해설"** 버튼 — 그 검사 한 건을 직전·또래 대비로 쉽게 풀어줌.

- `lib/aiReport.ts` `buildExamExplainContext()`: 검사 1건의 안축장 백분위 + 직전 검사 대비 증감 계산(기존 로직 재사용)
- `app/api/exam-explain`: Solar로 **핵심 내용별 불렛**(소제목+설명) 생성
- 안전: 정상범위·평균치 등 **미제공 수치 창작 금지**, 또래 비교는 안축장 백분위에만 한정(굴절은 값·변화만)
- UX 반복: 줄글→이모지→(이모지 남발 지적)→**깔끔한 불렛**으로 정착

## 2. 굴절 입력 부호 모델 정정 (정확도 기반 작업)
plus-cylinder 표기·큰 난시 등으로 **Sph가 +가 되는 처방**(예: Sph +1.00 / Cyl −4.00, SEQ −1.00)이 실재 → 기존 "Sph·Cyl 무조건 음수 저장" 모델은 광학적으로 부정확했음.

- **minus-cyl 표준 + 부호 있는 Sph**로 정정: Cyl은 음수 고정(minus-cyl), Sph는 ±토글로 실제 부호 저장
- `calcSeq` 실제 부호 기반(`Sph − Cyl크기/2`), 저장/편집 부호 보존
- 카드에 양수 Sph·SEQ는 `+` 표기(`fmtSigned`)
- SEQ는 표기 변환에 불변 → 기존 데이터·분석 영향 없음

## 3. 검사지 사진 OCR 자동입력 (AI 기능 3번)
장비가 다른 **안축장 검사지·굴절 검사지를 분리 업로드** → 각각 인식해 폼 자동 채움.

- `app/api/exam-extract`: **Upstage Information Extract**(`/v1/information-extraction`, model `information-extract`) — 검사지 종류별 JSON 스키마로 추출
  - axial: 안축장 R/L + 검사일 / refraction: S·C R/L(부호 포함) + 검사일
- `lib/examExtract.ts`: 사진 캔버스 축소(Vercel 본문 4.5MB 대비)→추출→**minus-cyl 정규화(plus-cyl이면 transpose)+부호 있는 Sph**로 폼 매핑
- 모달 상단 "검사지 사진으로 자동 채우기" 업로드 버튼 2개. **저장 전 사용자 검토 필수**(자동저장 안 함)
- 디버깅: IE는 `content`에 이미지 1개만 허용 / 1차 속성 type에 nullable 유니온 불가 → 단일 타입으로 수정
- 실제 검사지로 안축장·굴절 정확 인식 확인 ✅ (※ 페이지당 과금)
