## 4. 보안 3라운드 ★ (RLS 점검 → RPC 기반 접근제어)
실사용자 오픈 전 데이터 유출(아동 건강정보) 방지. 자세한 모델은 메모리 `project_security_model` 참고.

### R1 — 가장 시급한 구멍
- 초대코드 전체공개 SELECT/UPDATE 차단 → owner 본인 코드만
- 자녀 삭제 owner 전용 정책
- `is_guardian` search_path 보강

### R2 — 자녀별 초대 + 셀프추가/승격 차단
- **자녀별 초대코드**(`invite_codes.child_id`): 코드 1개=자녀 1명 → **형제 데이터 노출 방지**
- `create_child` / `create_invite_code` / `accept_invite_code` RPC화(SECURITY DEFINER)
- `child_guardians`: 넓은 ALL 정책 제거 → SELECT read-own만 (insert/update 차단)
- 비-owner 자녀 삭제 버튼 숨김
- **버그 수정**: `get_child_guardians`의 `user_id` 모호참조 → 보호자 목록 안 뜨던 문제 해결

### R3 — owner 양도
- `transfer_ownership` RPC + 설정 보호자 카드 "양도" 버튼 (owner→editor, 대상→owner)
- 보호자 제거도 `remove_guardian` RPC로 전환(RLS 정책 서브쿼리 이슈 회피)

### 핵심 교훈
RLS 정책 안에서 같은 테이블을 서브쿼리로 조회 금지 → SECURITY DEFINER 헬퍼(`is_guardian`/`is_owner`) 사용. (삭제 정책·보호자 목록 버그의 공통 원인)

## 6. 회원 탈퇴(계정 삭제) ★ — 개인정보처리방침 5조 이행
방침엔 "설정에서 데이터 삭제 가능"이라 명시돼 있었으나 **실제 기능 부재**(방침↔구현 불일치, 법적 리스크) → 구현.

- **`delete_account()` RPC**(SECURITY DEFINER, 단일 트랜잭션) — 자녀별 안전 처리:
  - 단독 소유자 → 자녀 + 모든 하위데이터(검사·케어·생활습관) 삭제
  - 공동 보호자 → 가장 먼저 등록된 보호자에게 소유권 양도 후 본인만 탈퇴(데이터 보존)
  - 편집자/열람자 → 본인 보호자행만 제거
  - 마지막에 프로필 + `auth.users` 삭제. 중간 실패 시 전체 롤백 → **데이터 유실 없는 안전한 실패**
- 설정 > 계정에 **"회원 탈퇴"** 버튼 + 확인 모달(소유권 양도 동작까지 안내)
- **버그 수정**: `eyebody_invite_codes`의 `owner_id`/`used_by`가 프로필을 FK 참조 → 프로필 삭제가 막힘. 프로필 삭제 전 초대코드 정리(내가 만든 코드 삭제, 수락한 코드는 `used_by`만 해제) 추가
- 실기기(폰) 탈퇴 테스트 통과 확인
- 파일: `docs/sql/2026-06-20-delete-account.sql`(신규), `queries.ts`(`deleteAccount`), `settings/page.tsx`(UI)

## 11. 온보딩 빈 상태 다듬기 (반복 → 정중·균형)
- **스플래시 → 앱 내 빈 상태**: 온보딩은 별도 라우트가 아니라 홈 탭의 `!activeChild` 빈 상태. 헤더 로고가 이미 있는데 큰 앱 아이콘 히어로를 또 띄워 **앱 아이콘 3중 노출**(로그인+헤더+온보딩) → 히어로 제거
- **카피·톤**: "환영합니다! / 내 아이의 근시, 체계적으로 관리해요" + "딱 3단계면 준비가 끝나요". 얼굴 이모지(👋→😊)는 시선 부담 있어 **제거**, 단계별 컬러가 온기 담당
- **균형 배치**: 위쪽 쏠림 → `flex justify-center min-h-[78vh]`로 세로 중앙, 카드 사이즈 확대
- **단계 아이콘**(컬러 타일, 어썸폰트 통일): 자녀 등록 `faChildReaching`(teal, text-3xl) / 기록하기 `faPen`(amber) / 변화 확인 `faChartLine`(rose). 세로로 긴 자녀 아이콘만 한 단계 크게
- **CTA**: `w-full` → 콘텐츠 너비(`w-fit`)+가운데 정렬로 버튼답게
- 파일: `components/onboarding/OnboardingFlow.tsx`
